# A 股投资侠客架构与边界

基础纪律见：

- [A 股投资侠客基础门规](xiake-rules.md)

## 目标定义

“A 股投资侠客”不是一个自动交易机器人，也不是一个只会吐出股票代码的模型。

它的目标是做一个个人投资决策助手：

- 每天巡场，观察市场温度和行业轮动。
- 从历史数据里寻找统计优势。
- 从新闻、公告、政策和舆情里识别机会与风险。
- 以 A 股为主战场，同时观察全球市场联动。
- 输出候选股票、理由、观察区、确认价、失效线和风险提示。
- 保持纪律，不替用户冲动下单。

一句话：

```text
用量化做内功，用新闻公告做耳目，用风控做侠规，用人工确认做最后一关。
```

## 两个项目的分工

### daozang-alpha

定位：

```text
离线量化研究引擎
```

它回答的问题是：

```text
从历史统计和模型角度看，哪些股票更可能有相对优势？
```

负责：

- Qlib 数据目录检查。
- 量价因子和标签构造。
- Alpha158、Alpha360 等基础特征集。
- LightGBM 等模型训练。
- IC、RankIC、分组收益和回测评估。
- 每日 alpha 分数导出。
- 模型版本、训练窗口、预测周期记录。

不负责：

- 盘中实时买卖判断。
- 直接抓取和理解新闻原文。
- 飞书推送。
- 自动下单。
- 最终投资建议。

核心产物：

```text
daozang-alpha/data/exports/alpha_scores_YYYY-MM-DD.csv
daozang-alpha/reports/backtest_*.html
daozang-alpha/reports/metrics_*.json
```

### beichen-alpha

定位：

```text
A 股每日候选池与执行前助手
```

它回答的问题是：

```text
今天这只票能不能看？有没有风险？是否接近可执行状态？
```

负责：

- 市场温度。
- 行业轮动。
- 动态股票池。
- 新闻和观点蒸馏。
- 巨潮公告风险。
- 宏观和政策事件。
- 盘中实时行情确认。
- 观察区、确认价、失效线、追高线。
- 飞书提醒。

不负责：

- 大规模模型训练。
- Qlib 实验管理。
- 深度回测框架。
- 长周期模型版本管理。

核心产物：

```text
beichen-alpha/data/watchlists/*.txt
beichen-alpha/data/runtime/realtime_checks.json
beichen-alpha/data/opinion_signals.jsonl
```

## 数据分层

### 第一层：行情和基础市场数据

来源：

- `chenditc/investment_data` 的 Qlib 格式 A 股数据包。
- 后续可补 AKShare、Tushare、BaoStock 等。

用途：

- `daozang-alpha` 训练、回测和打分。
- `beichen-alpha` 获取短线行情、市场温度、行业轮动。

边界：

- 可以用于个人研究原型。
- 不能盲信，需要抽样校验。
- 商业化前必须重新确认数据授权。

### 全球联动层：海外市场和宏观变量

A 股是主战场，但不能只看 A 股。

需要长期纳入观察的全球变量：

- 美股：纳斯达克、标普 500、费城半导体指数。
- 港股：恒生指数、恒生科技指数、中概股走势。
- 汇率：美元指数、USD/CNH、USD/CNY。
- 利率：美国 10 年期国债收益率、美联储议息和官员讲话。
- 商品：原油、黄金、铜、铁矿石、煤炭、锂、硅料等。
- 地缘：制裁、出口管制、战争、供应链扰动。
- 全球风险偏好：VIX、美元流动性、主要央行政策。

这些变量不直接替代 A 股信号，而是用来判断：

```text
今天 A 股的风险偏好、风格偏向、行业催化和外部压力。
```

例子：

- 美债利率上行，成长股和高估值科技可能承压。
- 费城半导体指数大涨，A 股半导体、AI 硬件、算力链可能受情绪传导。
- 原油上涨，油气、煤化工、航运、航空成本逻辑会变化。
- 人民币快速贬值，出口链和外资风险偏好都要重新评估。

边界：

- 全球变量先作为市场温度和行业偏向因子。
- 不因为单个海外变量直接给出买入结论。
- 进入模型前必须做滞后处理，避免未来函数。
- 要记录事件发生时间，区分盘前、盘中、盘后。

### 第二层：结构化事件数据

来源：

- 巨潮公告。
- 新闻源。
- 政策页面。
- RSS。
- 手工投喂的观点。

先由 `beichen-alpha` 负责抓取和蒸馏，形成结构化信号：

```json
{
  "date": "2026-07-04",
  "code": "600000",
  "event_type": "shareholder_reduce",
  "direction": "negative",
  "severity": 0.8,
  "decay_days": 10,
  "source": "cninfo"
}
```

后续 `daozang-alpha` 可以把这些信号转成日频因子：

```text
news_risk_3d
news_risk_7d
policy_boost_5d
has_negative_disclosure_10d
opinion_heat_3d
```

### 第三层：模型分数

由 `daozang-alpha` 输出：

```csv
trade_date,instrument,score,rank,pct_rank,model,feature_set,horizon_days,universe
2026-07-03,SH600000,0.0231,1,0.998,lightgbm,Alpha158,5,csi300
```

它只表达：

```text
模型认为这只股票在当前横截面中相对更有优势。
```

它不表达：

```text
今天必须买。
```

### 第四层：最终候选判断

由投资侠客决策层综合：

```text
最终候选分 =
  Qlib 模型分
  + 行业轮动分
  + 市场温度分
  + 新闻/观点正向分
  - 公告风险扣分
  - 追高风险扣分
  - 流动性/停牌/ST/涨跌停限制
```

最终输出应该是候选和纪律，而不是直接命令。

## 新闻和公告如何接入 Qlib

可以接，但不要第一步直接把新闻原文塞进 Qlib。

正确链路：

```text
新闻/公告原文
  -> beichen-alpha 蒸馏成结构化事件
  -> 事件按股票和日期聚合
  -> 转成日频特征
  -> daozang-alpha 做训练和回测
```

分阶段：

1. 第一阶段：新闻公告只做风控过滤。
2. 第二阶段：新闻公告进入最终综合评分。
3. 第三阶段：新闻公告转成 Qlib 因子，参与模型训练。

这样可以避免两个问题：

- 新闻文本噪声太高，先污染模型。
- 没有历史事件回放时，容易产生未来函数。

## 核心边界

### 不做自动交易

系统不连接券商账户，不自动下单，不自动撤单。

### 不承诺收益

任何模型分数、候选池、回测结果都只代表研究输出，不代表确定收益。

### 不让模型越权

模型只能给分，不能直接决定买卖。

买卖前必须经过：

- 风险过滤。
- 价格确认。
- 持有周期确认。
- 人工确认。

### 不用不可信回测说服自己

回测必须考虑：

- 交易成本。
- 滑点。
- 停牌。
- 涨跌停。
- ST。
- 新股上市天数。
- 退市样本。
- 股票池历史成分。

### 不把新闻原文当作干净因子

新闻、公告、政策、观点必须先结构化，再验证是否有增益。

## 开发阶段

### Phase 0: 数据可读

目标：

- 下载 Qlib A 股数据。
- 跑通 `doctor`。
- 确认目录结构和基础字段。

### Phase 1: 量价 baseline

目标：

- `Alpha158 + LightGBM`。
- 未来 5 日收益标签。
- CSI300 或 CSI500 股票池。
- 输出 IC、RankIC、分组收益。
- 导出每日 TopN alpha 分数。

### Phase 2: 执行前过滤

目标：

- `beichen-alpha` 读取 alpha 分数。
- 加入公告、新闻、行业、市场温度、实时价格。
- 输出更接近日常使用的候选池。

### Phase 3: 事件因子

目标：

- 把公告、新闻、政策、观点信号转成日频因子。
- 验证事件因子是否提高 IC 和分组收益。
- 无增益的事件因子不得进入主模型。

### Phase 4: 模拟盘和复盘

目标：

- 每天固定生成候选。
- 记录触发、未触发、止损、错过和实际表现。
- 让系统从“看起来对”变成“经得起日常复盘”。
