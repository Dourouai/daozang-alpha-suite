# 道藏 Alpha 可行性分析

## 你的想法

你想做一个 A 股投资股票推荐助手，并考虑用 Microsoft Qlib 作为底层框架。

这个方向是对的，但需要把目标拆清楚：

- 如果目标是“每天给出有依据的候选股票”，可行。
- 如果目标是“自动判断买卖并稳定赚钱”，现在不应该这样设计。
- 如果目标是“把规则经验、机器学习和风控合在一起”，Qlib 可以承担其中的机器学习和回测部分。

## 为什么 Qlib 合适

Qlib 的强项在于完整研究链路：

- 数据读取和缓存。
- 因子表达式。
- 数据集切分。
- 模型训练。
- 实验管理。
- 回测和绩效分析。

这正好覆盖了“先证明策略是否有统计意义”的部分。

## 最大不确定性

A 股推荐系统真正难的地方通常不在模型，而在数据和验证。

主要风险：

- 数据源质量不稳定。
- 复权口径不一致。
- 股票池存在幸存者偏差。
- 财报、公告、指数成分的历史时点容易穿越。
- 涨跌停、停牌、ST、流动性会让理论回测失真。
- 短线预测容易被交易成本和滑点吃掉。

因此，第一阶段不要急着做复杂模型，要先做“可信 baseline”。

## 建议定位

道藏 Alpha 应该定位为：

```text
离线研究和模型打分系统
```

而不是：

```text
全自动投资机器人
```

它每天输出：

- 股票代码。
- 模型分数。
- 分位排名。
- 所属股票池。
- 最近训练窗口。
- 预测周期。
- 风险标签占位。

是否进入观察池、是否执行、什么时候买，应该由更外层的风控和人工判断决定。

## MVP 验收标准

第一版不要看收益曲线漂不漂亮，而要看这些指标：

- IC 是否长期为正。
- RankIC 是否稳定。
- Top 分组是否显著优于 Bottom 分组。
- 加入交易成本后是否还有效。
- 换手率是否可接受。
- 不同年份表现是否过度集中。
- 市场下跌、震荡、上涨阶段是否都能解释。

如果这些都不过关，模型分数不能进入实际候选池。

## 技术路线

第一阶段：

- Qlib CN 数据目录检查。
- Alpha158 + LightGBM baseline。
- 5 日收益标签。
- CSI300 或 CSI500 股票池。
- 月度或季度滚动训练。
- 每日导出 Top50 分数。

第二阶段：

- 扩展股票池到全 A 可交易池。
- 加入 ST、停牌、涨跌停、流动性过滤。
- 接入 `beichen-alpha` 的风险事件、公告和新闻信号。
- 增加行业中性、风格暴露控制。

第三阶段：

- 多模型集成。
- 主题/行业轮动模型。
- 组合优化和仓位建议。
- 更严格的模拟盘记录。

## 当前结论

可以开始做，而且应该从 Qlib baseline 开始。

但项目成功的关键不是“用了什么模型”，而是：

- 数据是否可信。
- 回测是否没有穿越。
- 输出是否可解释。
- 风控是否比模型更保守。
