# A 股投资侠客基础门规

这些门规是系统的底层纪律。后续无论加 Qlib、新闻、公告、全球市场、LLM 还是自动化提醒，都不能越过这些边界。

## 第一条：主战场是 A 股，但不闭门看 A 股

系统主攻 A 股候选池，但每天必须观察全球市场联动。

基础观察项：

- A 股指数、成交额、涨跌家数、涨跌停。
- 港股和中概股。
- 纳斯达克、标普 500、费城半导体指数。
- 美债利率、美元指数、人民币汇率。
- 原油、黄金、铜、铁矿石等商品。
- 重要央行、地缘、出口管制和产业政策事件。

全球变量只用于判断风险偏好、风格偏向和行业外部压力，不能因为单一海外信号直接给出买入结论。

## 第二条：先看风险，再看机会

任何候选股票进入推荐前，先过风险门。

硬风险包括：

- ST、退市风险、重大处罚、重大诉讼。
- 明确减持、预亏、财务异常、监管问询。
- 停牌、流动性不足、极端缩量。
- 涨停无法买入、跌停无法退出。
- 新股和次新股样本不足。

如果风险门不过，模型分数再高也只能降级或剔除。

## 第三条：模型只能给分，不能下令

`daozang-alpha` 的模型输出只代表相对统计优势。

模型可以说：

```text
这只股票在当前横截面中分数较高。
```

模型不能说：

```text
今天必须买。
```

最终候选必须经过：

- 模型分数。
- 市场温度。
- 行业轮动。
- 新闻公告风险。
- 实时价格确认。
- 人工确认。

## 第四条：不用不可信数据训练信念

数据不干净，模型越复杂越危险。

每个数据源都要记录：

- 来源。
- 下载时间。
- 更新频率。
- 字段含义。
- 是否复权。
- 是否包含退市样本。
- 是否存在延迟或缺失。

关键字段必须抽样检查：

- 开高低收。
- 成交量和成交额。
- 前复权/后复权。
- 停牌。
- 涨跌停。
- 指数成分。

## 第五条：绝不使用未来函数

任何特征进入模型前，都必须确认它在当时真实可见。

特别注意：

- 财报发布日期，不是财报期末日期。
- 公告发布时间，不是事件标题日期。
- 指数成分调整生效日。
- 新闻发布时间和交易时区。
- 海外市场盘前、盘中、盘后的时间对齐。

如果无法确认时点，先不用作训练因子。

## 第六条：回测先保守，再谈漂亮

回测必须默认保守。

至少考虑：

- 交易成本。
- 滑点。
- 停牌。
- 涨跌停。
- T+1。
- 换手率。
- 流动性容量。
- 样本外年份。

不能只看收益曲线，还要看：

- IC。
- RankIC。
- 分组收益。
- 最大回撤。
- 年度稳定性。
- 市场状态分层表现。

## 第七条：候选不是买入，确认不是追高

系统输出的是候选池和纪律，不是冲动指令。

每只候选至少要有：

- 推荐理由。
- 模型分数或规则分。
- 观察区。
- 确认价。
- 失效线。
- 追高线。
- 风险标签。
- 预期持有周期。

站上确认价不等于可以无限追。超过追高线，宁可错过。

## 第八条：短线更重纪律，长线更重逻辑

不同周期不能混用规则。

超短线关注：

- 市场情绪。
- 板块共振。
- 量价确认。
- 追高风险。
- T+1 风险。

中短线关注：

- 行业轮动。
- 事件催化。
- 资金持续性。
- 回撤和止损。

中长期关注：

- 基本面趋势。
- 估值位置。
- 产业周期。
- 政策方向。
- 财报兑现。

## 第九条：不自动下单，不承诺收益

系统不连接券商账户，不自动下单，不自动撤单。

所有输出只作为个人研究候选，不构成投资建议，不承诺收益。

## 第十条：每日留痕，定期复盘

系统每天生成的候选、过滤原因、风险提示和实际表现都要留痕。

必须记录：

- 当日候选。
- 被剔除股票和剔除原因。
- 是否触发确认价。
- 是否超过追高线。
- 是否触发失效线。
- 1 日、3 日、5 日、10 日后表现。
- 人工是否执行，以及执行理由。

复盘要同时保存成功和失败样本。只保存成功案例，会把系统带偏。

## 第十一条：先小闭环，再大野心

开发顺序必须克制：

1. 数据可读。
2. baseline 可跑。
3. 回测可信。
4. 分数可导出。
5. 候选可解释。
6. 风险可过滤。
7. 复盘可追踪。
8. 再考虑复杂模型和自动化。

每一步都要能被验证，不能只靠感觉。

## 第十二条：系统要帮助人冷静

投资侠客不是为了让人更兴奋，而是让人更清醒。

好的输出应该让用户知道：

- 为什么这只票值得看。
- 为什么现在还不能动。
- 什么情况说明判断错了。
- 什么位置以后就不追了。
- 什么时候应该放弃。

宁可少推荐，也不要乱推荐。
