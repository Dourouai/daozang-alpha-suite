# 宏观事件驱动因子

宏观事件因子用于回答一个问题：今天外部宏观事件会把 A 股的短线偏好推向哪个方向？

它不直接推荐股票，只给行业和主题加减分。最后仍要经过大盘环境、行业共振、个股强弱、流动性、风险日历和实时站稳确认。

## 数据分层

| 层级 | 内容 | 获取方式 | 当前接入 |
|---|---|---|---|
| 确定性行情 | 纳指、标普、费城半导体、美元指数、美债收益率、黄金、原油、日经、韩国 KOSPI/KOSDAQ、恒生科技、富时 A50 | AKShare、Yahoo/Stooq、交易所或指数公开页 | 规划中 |
| 宏观 RSS | Fed 讲话/政策新闻、BEA 新闻发布、BLS 就业和通胀 RSS | 官方 RSS | 已接入，低/中置信度自动分类 |
| 官方政策页 | 财政部、发改委、人民银行等政策列表页标题 | 官方公开页面 | 已接入，低/中置信度自动分类为 `policy_event` |
| 宏观日历 | 非农、失业率、CPI、PCE、ISM、FOMC、点阵图 | BLS、BEA、ISM、Federal Reserve、财经日历 | 规划中，负责“是否超预期” |
| 语义事件 | 美联储主席或官员讲话偏鹰/偏鸽、地缘风险、AI资本开支变化、芯片出口管制 | 官方讲话、新闻摘要、个人观点源蒸馏 | RSS 已接入，CSV 可人工覆盖 |
| 个人观点 | 博主/视频号对宏观事件和行业映射的解释 | `ingest` 蒸馏为观点信号 | 已接入观点源 |

## 事件枚举

事件先分三类：

| 策略类型 | 作用 | 适合放在哪里 |
|---|---|---|
| 央行/流动性 | 决定风险偏好和估值压力，例如加息、降息、降准、净投放、美元和美债变化 | 候选层加减分，早盘执行层做宏观同步/冲击 |
| 政策事件 | 决定财政、税费、产业、资本市场、监管、流动性等国内政策方向 | 只做行业偏向，不直接生成买入；政策标题源置信度低于公告和确定性数据 |
| 重点事件 | 决定避险、制裁、重大会议等短线方向 | 负面事件可重扣分，正面事件只给行业加分，仍需板块确认 |
| 产业创新 | 决定新主题是否进入景气验证，例如 AI资本开支、半导体突破、机器人量产、固态电池产业化 | 候选层提高行业优先级，执行层必须等放量和板块共振 |

## 加策略的方法

1. 先定义事件，不直接定义股票。

```text
事件：全球主要央行加息
含义：流动性收紧，成长估值承压
行业：银行/公用事业相对加分，半导体/AI硬件/创新药扣分
有效期：1-2天
执行要求：即使候选分提升，也必须等板块同步和个股站稳
```

2. 再决定它是加分、扣分还是一票否决。

```text
普通流动性事件：+/- 4 到 8 分
重大政策支持：+6 到 +10 分，但不直接可买
重大制裁/诉讼/监管处罚：交给风险日历或公告源，一票否决
产业创新：+5 到 +8 分，必须叠加行业轮动和放量确认
```

3. 最后设置有效期。

```text
央行讲话/隔夜外盘：0.5-1天
就业/通胀/美债美元：1-2天
政策会议/产业政策：2-5天
产业创新订单/量产：3-10天，但每天递减
```

## CSV 格式

自动 RSS 源默认读取：

```text
config/macro_rss_feeds.csv
```

当前默认启用：

- Fed Speeches: `https://www.federalreserve.gov/feeds/speeches.xml`
- Fed Monetary Policy Press: `https://www.federalreserve.gov/feeds/press_monetary.xml`
- BEA News Releases: `https://apps.bea.gov/rss/rss.xml`

BLS RSS 模板也在配置里，但默认关闭。有些网络环境下 BLS 会对脚本访问返回 403，后续更适合接 BLS API 或下载固定新闻稿页面。

官方政策页默认读取：

```text
config/macro_policy_pages.csv
```

当前默认启用：

- 财政部政策发布：财政、税费、补贴、贴息、专项债等政策线索
- 国家发改委通知：产业政策、设备更新、先进制造、基建、价格监管等线索
- 人民银行新闻：货币政策、流动性、金融支持等线索

政策页只解析标题、日期和链接，所以默认置信度低于手动确认事件。真正重要的政策可以在 `config/macro_events.csv` 中手动启用一条 `policy_event` 进行覆盖。

人工覆盖源默认读取：

```text
config/macro_events.csv
```

CSV 字段：

```csv
enabled,date,title,source,event_type,stance,positive_sectors,negative_sectors,base_score,decay_days,confidence,detail,url
```

示例：

```csv
true,2026-07-03,美国就业弱于预期,manual,us_jobs,dovish,"黄金/有色/半导体/AI硬件/医药","银行/煤炭",8,2,0.80,降息预期升温,
```

评分规则：

- `enabled=false` 的行不会生效，适合保留模板。
- `base_score` 建议 4-8，极少超过 10。
- `decay_days` 越短，说明事件越适合盘前/早盘，不该影响太久。
- `confidence` 是事件判断置信度，例如讲话语义不确定可以设为 `0.6`。
- 同一事件只按行业匹配加一次分，不因为股票同时命中多个主题而重复加分。

关闭 RSS，只保留 CSV 覆盖：

```bash
PYTHONPATH=src python3 -m beichen_alpha --disable-macro-rss
```

关闭官方政策页：

```bash
PYTHONPATH=src python3 -m beichen_alpha --disable-policy-pages
```

## 模型位置

```text
候选评分 = 大盘环境 + 宏观事件 + 风格偏向 + 行业共振 + 个股强弱 + 流动性 + 观点偏向 + 基本质量 - 风险扣分

执行评分 = 实时站稳 + 放量确认 + VWAP + 板块同步 + 宏观同步 - 追高惩罚 - 周五/T+1惩罚
```

宏观事件是方向盘，不是油门。它只能提高某个行业值得看的概率，真正执行仍要看 10:00-10:30 的站稳、放量、VWAP 和板块同步。
